### 3. Key Design Decisions

- **ADC2_ChannelCallback** stays in `main.c` for visibility — calls `MC_APP_ISR()` + `X2Cscope_Update()`
- **`mcInitDone` guard** prevents ISR from executing FOC before initialization completes
- **Direction reversal**: SW2 stops motor (FOC decelerates to zero via `MCAPP_STOPPING` state), then auto-restarts in new direction
- **X2Cscope**: kept existing T1S TCP/IP implementation (port 12666), not the old UART one from LAB4_FOC
- **MCC generated files**: NEVER modified directly — if changes needed, user regenerates via MCC

### 4. Overcurrent Protection (CMP3 + PWM Fault PCI)

Implemented in `MC_HAL_OverCurrentProtectionInit()` inside `mc_hal.c`:

- **CMP3/DAC3**: Monitors bus current via OPA3 output on **CMP3A (RA5)** — `INPSEL = 0`
- **DAC reference**: `CMP_REF_DCBUS_FAULT` ≈ 2979 counts (10A threshold)
- **Comparator polarity**: `CMPPOL = 0` (non-inverted: output HIGH when overcurrent)
- **PWM Fault PCI**: `PGxF1PCI1 = 0x03001000` (TERM=1 cycle-by-cycle, ACP=3 latched, PPS=0)
- **PCI source**: `PGxF1PCI2 = 0x40000000` (bit 30 = CMP3 output)
- **Generators disabled** before writing PCI registers, then re-enabled
- **DAC calibration**: reads Flash at 0x7F20E0 for POSINLADJ/NEGINLADJ/DNLADJ
- **Hysteresis**: 45 mV (HYSSEL=3)

**Status**: INPSEL was corrected from 3 (CMP3D) to 0 (CMP3A) at end of session. Needs testing to confirm cycle-by-cycle clipping works.
